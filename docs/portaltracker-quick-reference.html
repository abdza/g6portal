üèóÔ∏è System Overview

PortalTracker is a dynamic form and workflow management system that creates database tables and user interfaces on-the-fly based on configuration.

Key Components

  • PortalTracker - Main configuration
  • PortalTrackerField - Field definitions
  • PortalTrackerStatus - Workflow states
  • PortalTrackerTransition - State changes
  • PortalTrackerData - Excel import/export

Database Tables Created

  • trak_{module}_{slug}_data - Main data
  • trak_{module}_{slug}_updates - Audit trail
  • Dynamic columns per field definition
  • Automatic indexing for search/filter fields

üìä Tracker Types

Tracker

Full Workflow System

  • Status transitions
  • Complete audit trail
  • Role-based permissions
  • Email notifications
  • File attachments

Statement

Role-Based Data

  • Role-based access control
  • Basic record status
  • No audit trail
  • Simpler permissions

DataStore

Simple Storage

  • Pure data storage
  • No roles or workflows
  • Direct data access
  • High performance

‚ö†Ô∏è Choosing the Right Type

Use DataStore for simple data, Statement for role-based access, and Tracker for full workflow requirements. Choose the minimal type needed for better performance.

üîí Security Features

  • Audit Logging - All operations logged to PortalErrorLog

üîß Common Issues & Solutions

‚ö†Ô∏è Table Not Created

Symptom: "Table doesn't exist" errors

Solution: Call tracker.updatedb(dataSource) or save the tracker

‚ö†Ô∏è Field Missing from DB

Symptom: New fields not in queries

Solution: Save the PortalTrackerField to trigger schema update

‚ö†Ô∏è Role Access Denied

Symptom: Users can't see expected data

Solution: Check UserRole assignments and Data Compare rules

‚ö†Ô∏è Excel Import Failing

Symptom: No data imported or errors

Solution: Check field mappings, data types, header row config
